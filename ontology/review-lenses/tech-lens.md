# Tech Lens

## Focus

Evaluate whether the event is backed by real system capability rather than descriptive language or prompt-only behavior.

## Use This Lens For

Use this lens when reviewing implementation claims, runtime behavior, or infrastructure changes that must hold under load and failure.

## Not This Lens For

Do not use this lens for product strategy, adoption, or brand questions unless there is a concrete technical claim to verify.

## Questions

- Is this structural capability or prompt assembly?
- Do context, tools, state retention, latency, and cost support the promise with measurable evidence?
- What are the failure modes?
- Is the handoff contract traceable?
