# User Reality Lens

## Focus

Evaluate whether the event matches observed user behavior, language, and friction in real use.

## Use This Lens For

Use this lens when reviewing adoption claims, user complaints, or reported workflow changes that need validation in actual behavior.

## Not This Lens For

Do not use this lens for internal roadmap debates unless they depend on user evidence or customer-facing signals.

## Questions

- What user evidence supports or weakens the adoption claim?
- Where is the friction?
- Are complaints surface-level or signs of a deeper mental model break?
- Does user language align with official framing?
