# Product Lens

## Focus

Evaluate whether an event changes the product's structure, not just its surface feature set.

## Use This Lens For

Use this lens when reviewing events that may alter product architecture, ownership, or adoption mechanics.

## Not This Lens For

Do not use this lens for copy edits, UI polish, or isolated feature requests that leave structure unchanged.

## Questions

- What layer did this event actually change?
- What problem is being addressed?
- Is this a feature increment or a structural migration?
- What is the strongest evidence for the timing, and what is the strongest competing explanation for doing it now?
- Does this change control boundary, responsibility allocation, or adoption logic?
