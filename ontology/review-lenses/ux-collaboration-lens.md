# UX Collaboration Lens

## Focus

Evaluate human-agent collaboration quality, capability visibility, intervention timing, and recovery design.

## Use This Lens For

Use this lens when reviewing interactions where a user and system share responsibility, state, or decision-making.

## Not This Lens For

Do not use this lens for purely visual polish or backend capability changes that do not affect collaboration behavior.

## Questions

- What state cues make the current system or agent state visible, and what signals show where the capability boundary starts and ends?
- What triggers should cause the system to clarify, and what triggers should cause it to act?
- Is there a recovery path?
- Are high-risk actions correctly gated?
- What visible signals, confirmations, or recovery paths support the trust calibration judgment?
- Is there a temporary or task-specific interface for complex collaboration?
