## Core Position

This is a review system, not a multi-persona conversation.

## Review Targets

Review targets are `Claim`, `Pattern`, and `Thesis`.

## Review Constraints

- Reviewers cannot add facts.
- Review output may only re-rank, support, weaken, refute, or request more evidence for existing objects.
- Verdicts can support, weaken, refute, or request more evidence.

## Judge Authority

The Judge is the only role allowed to promote `Claim -> Pattern` or downgrade `Claim -> WhyNowHypothesis`.

## Review Protocol

1. Independent first pass.
   Each seat reads the same evidence and records its own judgment, weak points, and missing evidence.
2. Cross-examination.
   Seats challenge one another's assumptions, but cannot add new facts.
3. Judge synthesis.
   The Judge reconciles support, weakness, refutation, and evidence gaps into a single verdict.
4. Preserved disagreements.
   Any unresolved conflict stays on record instead of being forced into premature agreement.

## Seat Model

The review process can still use multiple seats for critique, but each seat must stay within evidence already on record.

### Typical Seats
- Product seat
- Design seat
- Technical seat
- User reality seat
- Leadership / business seat
- Brand / positioning seat
- Contrarian seat
- Judge seat

## Review Output

Each review should end with:
- supported
- weakened
- refuted
- needs more evidence
- promoted
- downgraded
- preserved disagreement
